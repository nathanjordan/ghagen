## [0.4.0] (unreleased)

### ⚠ BREAKING CHANGES

- **Models are no longer frozen.** `Object.isFrozen(step({...}))` now returns `false`. `_data` and `_meta` are runtime-mutable so synthesis-time transforms (e.g. `PinTransform`) can rewrite fields on deep clones. TypeScript types retain `readonly` modifiers, so existing TS consumers are unaffected.
- **`Model` gains `_sourceLocation`.** Every model now carries a `SourceLocation | null` captured via stack walking at factory call time. Code doing deep equality on raw models will see the new field.
- **New `"bin"` entry registers a `ghagen` CLI command** when installed globally or via `npx`.
- **Default YAML header changed** from `Auto-generated by ghagen` to `This file is generated by ghagen from {source_file}. Do not edit manually.`

### Features

- **App orchestration** — `new App({ root, header, lockfile, transforms })` with `addWorkflow()`, `addAction()`, `synth()`, `check()`. CDK-inspired register-then-synthesize pattern.
- **CLI** — `ghagen synth`, `check-synced`, `lint`, `init`, `deps pin`, `deps check-synced`, `deps upgrade`. Built on commander.
- **Pin subsystem** — lockfile-based action version pinning (`readLockfile`, `writeLockfile`, `pinTransform`). Shared `.github/ghagen.lock.toml` format with the Python package for cross-language interop. Includes `resolveRef`, `listTags`, and version comparison utilities.
- **Lint engine** — 4 built-in rules (`missing-permissions`, `unpinned-actions`, `missing-timeout`, `duplicate-step-ids`). Configurable via `.github/ghagen.toml` or `package.json`. Three output formats: human, JSON, GitHub Actions annotations.
- **Config loading** — reads `[options]` from `.github/ghagen.toml` or `package.json` `"ghagen"` field. Supports `auto_dedent` toggle.
- **Auto-dedent** — `step({ run: ... })` strings are automatically dedented (matching the Python package). Controlled by `setAutoDedent()` or config.
- **`cloneModel()`** — deep-clone models preserving Symbol brands and function references (avoids `structuredClone` limitations).
- **Config-file entry-point discovery** — uses `jiti` to run `.ts`/`.js` config files without a build step.
- **Comprehensive exports** — `App`, transforms, pin, lint, config, paths, and dedent utilities all exported from `@ghagen/ghagen`.
